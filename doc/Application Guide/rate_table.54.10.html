Reducing Rating Management Components

Rate books, rate table definitions, rate routines, and parameter sets are rating management components. This topic describes how you can reduce rating management components in your implementation.

Apply Rating Management Components to More Than One Policy Line

Rating Management components apply to a specific policy line, such as the Personal Auto line of business. Alternatively, these components can apply to more than one policy line when you set the Policy Line to Generic. In code, a generic policy line is null.
Line-specific Rating Management components can reference generic Rating Management components as well as components defined for the same policy line. For example, a rate book with Personal Auto as its policy line can reference generic and Personal Auto rate tables. The Personal Auto rate book cannot reference Commercial Property rate tables.
Generic Rating Management components can only reference generic components. They cannot reference Rating Management components defined for a specific policy line. For example, a rate book with a generic policy line can reference generic rate tables only. The generic rate book cannot reference Commercial Property rate tables.

Combine Similar Rate Routines and Parameter Sets

Parameter set wrappers enable you to combine similar parameter sets and rate routines.
You may have parameters sets that are essentially the same, except that the coverage changes by policy line, for example. With wrappers, a single parameter set can apply to more than one policy line. For example, a single parameter set can uses a wrapper to select the coverage by policy line or other characteristic of the policy. Wrappers are implemented in Gosu code.
In addition, rate routines that are essentially the same can be combined into a single rate routine by using a parameter set with wrappers.
